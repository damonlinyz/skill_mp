# Skill Distiller - 多源文档读取策略

## 任务

高效读取用户提供的多源内容（飞书文档链接 + 图片），为后续的方法抽取和格式抽取做准备。

---

## 读取策略

### 并行读取原则

- **飞书文档**：多个链接同时发起读取，不串行等待
- **图片**：与其他内容并行处理
- **用户补充描述**：与文档读取同步进行（不需要等待文档读完）

### 内容优先级

| 优先级 | 内容类型 | 说明 |
|-------|---------|------|
| 1 | 飞书文档文字内容 | 核心分析方法和结论的来源 |
| 1 | 飞书文档中的表格 | 数据细节和计算逻辑 |
| 2 | 文档中的图片截图 | 表格截图、流程图、架构图 |
| 3 | 用户补充描述 | 确认分析目的和场景 |
| 4 | 其他附件（如有） | PDF、PPT 等 |

---

## 飞书文档读取

### 读取模式选择

当用户提供飞书链接时，选择读取方式：

```
飞书链接检测到，请选择读取方式：

  [1] 浏览器方案（推荐）
      复用本机 Chrome 登录态
      ✅ 内部文档、需要权限的文档都能读
      ✅ 无需配置 token
      ⚠️  需要本机安装 Chrome + playwright

  [2] MCP/App Token 方案
      通过飞书 API 调用
      ✅ 稳定，不依赖浏览器
      ⚠️  需要先配置 App ID / App Secret
      ⚠️  内部文档需要管理员给应用授权

选择 [1/2]（未指定则默认 1）：
```

### 内容提取要求

对每个飞书文档，提取以下信息：

```
## 文档 {N}：{文档标题}
**来源**：{链接}
**类型**：{报告/方案/研究/综合}
**规模**：{字数/页数/表格数/图片数}

### 文字内容
{按段落提取，保持结构}

### 表格内容
| 表名 | 行列数 | 内容概要 |
|-----|-------|---------|
| {表1} | {RxC} | {描述} |
| {表2} | {RxC} | {描述} |

### 图片/截图
- [图片1]：{位置}，{内容描述}，待 vision 处理
- [图片2]：{位置}，{内容描述}，待 vision 处理
```

## 图片理解

### 需要理解的内容

以下图片必须用 vision 能力理解：

| 图片类型 | 理解目标 |
|---------|---------|
| **表格截图** | 行列结构、数据内容、汇总行、计算公式 |
| **流程图/架构图** | 步骤顺序、分支逻辑、循环关系、角色/系统 |
| **分析结果图** | 图表类型（折线/柱状/饼图/散点）、数据趋势、坐标含义 |
| **PPT/文档截图** | 页面布局、文字内容、章节结构 |
| **方法说明图** | 使用了什么方法（归因/漏斗/对比等）|

### Vision 处理输出

```
## 图片 {N}：{图片描述}
**类型**：{表格/流程图/结果图/PPT截图/其他}
**理解结果**：

### 内容结构
{结构化描述}

### 关键信息
- {关键点1}
- {关键点2}

### 推断内容（🔶推断）
- 🔶{推断内容}：依据 {...}

### 置信度
- ✅ 明确内容：{比例}
- 🔶 推断内容：{比例}
```

## 内容合并与去重

### 合并规则

| 情况 | 处理方式 |
|-----|---------|
| 同一内容文字版 vs 图片版 | 优先文字版，图片版作为补充 |
| 多个文档有相同章节 | 保留第一个，其余标注"同文档X" |
| 结论在多个地方重复 | 取最完整的版本，标注来源 |
| 表格数据在文字中复述 | 只保留表格，文字标注"见上表" |

### 冲突处理

| 冲突类型 | 处理方式 |
|---------|---------|
| 方法说法不一 | ❓存疑，列出各文档版本 |
| 结论不一致 | ❓存疑，列出各文档版本及依据 |
| 格式要求不一致 | ❓存疑，取最通用的格式 |

### 来源标注

所有抽取内容必须标注来源：

```
{抽取内容}
**来源**：{doc1 / doc2 / img1 / img2 / 用户描述}
```

## 读取质量问题

### 常见问题及处理

| 问题 | 处理方式 |
|-----|---------|
| 文档需要权限 | 提示用户提供有权限的链接，或换用浏览器方案 |
| 表格图片无法提取数据 | 使用 vision 理解，标注🔶推断 |
| 文档结构混乱 | 保持原结构提取，标注"结构较乱，需理解后重组织" |
| 内容跨多页/多章节 | 完整读取所有部分，不截断 |
| 外部链接/引用 | 记录链接，不展开读取（除非用户要求）|

### 无法读取时的处理

当文档无法读取时：

```
⚠️ **文档读取失败**：{文档标题}
**链接**：{链接}
**原因**：{权限不足/格式不支持/网络问题/其他}

**建议**：
1. 确认链接是否正确
2. 尝试使用浏览器方案（可能需要登录）
3. 或者提供其他有权限的文档链接

可继续处理已成功读取的 {N} 个文档，其他文档稍后补充？
```
