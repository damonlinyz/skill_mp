# Skill Distiller - Skill 草稿生成 Prompt

## 任务

基于前几步的抽取结果（内容理解 + 方法抽取 + 格式抽取），按照 skill_mp 的标准 Schema 生成 Skill 草稿。

---

## 输入材料

### 已完成的工作：

1. **多源内容**：{N} 个飞书文档 + {M} 张图片 的内容理解结果
2. **分析方法**：明确方法 {N} 个 + 推断方法 {N} 个
3. **输出格式**：文档结构 + 结论呈现 + 建议格式
4. **Schema 补全**：推断的 TRIGGER、Red Flags、统计陷阱

### 置信度汇总：

```
✅ 明确（可直接使用）：{N} 项
🔶 推断（需平台校验）：{N} 项
❓ 存疑（需人工确认）：{N} 项
```

---

## 生成原则

### Schema 完整性

skill_mp 的标准 Schema 必须包含：

```
---
name: {skill-name}
description: {一句话描述}
---

# {Skill 标题}
## TRIGGER when
## DO NOT TRIGGER when
## Overview
## 完整分析流程
## [各步骤详细说明]
## 结构化输出
## 常见错误
## Red Flags - 停下来检查
## 统计陷阱警示
```

### 草稿质量要求

| 要求 | 说明 |
|-----|------|
| **结构清晰** | 层次分明，步骤之间有逻辑关系 |
| **可执行** | 每步都有明确的动作指导 |
| **格式可用** | 输出格式模板可直接使用 |
| **标注置信度** | 🔶推断和❓存疑的内容必须标注 |

### 草稿 vs 终稿的差异

| 部分 | 草稿处理 | 终稿期望 |
|-----|---------|---------|
| name | 自动生成，平台测试后可能调整 | 业务确认 |
| TRIGGER | 🔶推断，基于目的反推 | 业务补充具体场景 |
| 推断方法 | 🔶推断，平台测试时校验 | 确认/修正/补充 |
| 统计陷阱 | 🔶推断，参考通用模式 | 业务补充具体案例 |
| 置信度 | 必须标注 | 平台测试后消除 |

## 生成模板

### Skill 名称生成

基于分析目的自动生成：

```
## 名称生成规则

{domain}-{analysis-type}-skill

示例：
- intelligent-data-analysis（智能数据分析）
- cost-attr-analysis（成本归因分析）
- sales-trend-analysis（销售趋势分析）
- user-portrait-analysis（用户画像分析）

说明：
- domain：业务领域或指标类型
- analysis-type：分析方法类型
- 优先使用已知的通用命名，无则新创
```

### SKILL.md 骨架

```markdown
---
name: {skill-name}
description: {一句话描述，30字以内，说明适用于什么场景}
version: "0.0.1-draft"
---

# {Skill 标题}

## TRIGGER when

- 用户说/问"帮我分析{具体指标/现象}"
- 用户说/问"看看{问题}的原因"
- 用户说/问"{分析目的}，帮我分析"
- 需要对{分析类型}进行分析时

## DO NOT TRIGGER when

- 用户上传了 Excel/CSV 文件要求分析 → 用 table-data-analysis
- 用户上传了表格文件 → 用 table-data-analysis
- 用户只是想问指标数据 → 直接查询平台
- {其他不适用场景}
