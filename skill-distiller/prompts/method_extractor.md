# Skill Distiller - 分析方法抽取 Prompt

## 任务

从已读取的文档内容中，识别和抽取：
1. **明确方法**：文档中直接提及的分析方法
2. **推断方法**：文档中没有直接写，但可以从分析步骤中推断的方法
3. **分析流程**：从步骤顺序中还原分析流程

---

## 方法识别框架

### 通用分析方法清单

使用以下清单识别文档中的方法：

| 方法 | 核心特征 | 识别关键词 |
|-----|---------|-----------|
| **指标归因** | 拆解为子指标，计算各子指标贡献 | "贡献度"、"拆解为"、"子指标"、"构成" |

## 明确方法抽取

### 抽取规则

直接在文档中识别到的方法，标注为 ✅明确：

```
### ✅ 明确方法

| # | 方法名 | 文档表述 | 所在位置 | 置信度 |
|---|--------|---------|---------|--------|
| 1 | 指标归因 | "将成本拆解为人力成本、租金成本..." | doc1 第3段 | ✅明确 |
| 2 | 维度归因 | "按区域拆分，华东区贡献60%" | doc1 第5段 | ✅明确 |
```

### 抽取要点

- **原文引用**：尽量引用文档中的原话
- **位置标注**：doc编号 + 段落/表格/图片位置
- **方法确认**：确保是"分析方法"而非"数据处理方式"

## 推断方法抽取

### 推断策略

当文档没有直接写方法时，从以下线索推断：

| 推断线索 | 可能的隐含方法 | 推断依据 |
|---------|--------------|---------|
| 有子指标数据但没说归因 | 指标归因 | 子指标+汇总指标的组合出现 |
| 有维度拆分但没说维度归因 | 维度归因 | 按维度分组统计出现 |
| 有 Top N 排序 | 贡献计算 | 按贡献度排序的列表 |
| 有汇总→明细的层次 | 下钻分析 | 层级钻取路径 |
| 有分组对比但无统计检验 | 对比分析 | 分组数据对比，无p值 |
| 有图表无统计检验 | 描述性分析 | 图表展示为主 |
| 有预测但无模型说明 | 趋势预测 | 基于历史趋势的预测 |

### 🔶推断标注格式

```
### 🔶 推断方法

| # | 方法名 | 推断依据 | 置信度 | 存疑点 |
|---|--------|---------|--------|--------|
| 1 | 指标归因 🔶推断 | doc2中有"子指标"列表+汇总数据，但未写"归因" | 中 | 文档未明确说明计算方式 |
| 2 | 下钻分析 🔶推断 | doc1流程图显示 区域→城市→站点 路径 | 高 | 直接可确认 |
```

### 置信度判断

| 置信度 | 判断标准 |
|-------|---------|
| **高** | 有明确的步骤/数据结构支撑，几乎可以确认 |
| **中** | 有线索但不够直接，需要平台测试确认 |
| **低** | 推断成分较大，可能有其他解释 |

## 分析流程还原

### 流程识别

从文档的步骤顺序中还原分析流程：

```
### 分析流程（从文档中还原）

**步骤顺序**：
1. [doc1 第2段] 获取指标元数据 → 确定分析指标
2. [doc1 第3段] 拆解为子指标 → 指标归因
3. [doc1 第5段] 按区域维度拆分 → 维度归因
4. [doc2 图1] 发现异常区域 → 下钻到城市级别
5. [doc1 第8段] 计算各因素贡献度 → 贡献计算
```

## 特殊情况处理

### 混合方法

当文档包含多种分析方法时：
- 全部列出，不遗漏
- 标注各方法的主次关系

### 方法链

当分析方法有顺序依赖时：
- 还原完整的方法链
- 标注关键决策点

### 缺方法

当文档内容不足以识别方法时：
- 标注"方法不明确"
- 从目的反推可能的方法（标注🔶推断）

## 注意事项

| 注意点 | 说明 |
|-------|------|
| **宁缺毋滥** | 推断方法要标注🔶推断，不确定的不硬凑 |
| **来源可溯** | 每个方法都要标注来源文档 |
| **区分方法与目的** | "分析成本增加原因"是目的，"拆解子指标"是方法 |
| **区分方法与工具** | "用 SQL 查询"是工具，"维度归因"是方法 |
| **区分方法与输出** | "给出 Top N 排名"是输出格式，"贡献计算"是方法 |
