# Skill Distiller - 用户输入确认 Prompt

## 任务

确认用户要蒸馏的 Skill 的基本信息，包括分析领域、分析目的、文档来源。

---

## 确认问题序列

### Q1：分析领域确认

**问题**：
```
检测到您要蒸馏 Skill，请问这是什么领域的分析？

示例：
- 业务分析（销售/运营/市场）
- 数据分析（指标归因/趋势/用户）
- 技术分析（性能/架构/代码）
- 研究综述（竞品/技术选型）
- 其他：{请描述}

（可以直接说"业务分析"或具体描述，我会推断）
```

### Q2：分析目的确认

**问题**：
```
这个分析的核心目的是什么？

示例：
- 归因分析（找原因、拆解因素）
- 趋势分析（看变化、预测走势）
- 探索分析（找规律、发现异常）
- 评估分析（评效果、比方案）
- 综合分析（包含多种类型）

（如果不确定，我会从文档内容中识别）
```

### Q3：文档来源确认

**问题**：
```
您的分析文档来自哪里？

  [A] 飞书文档链接
      直接提供飞书文档链接（支持多个）

  [B] 上传图片/截图
      上传包含分析结果的截图

  [C] 飞书链接 + 图片
      同时提供飞书链接和图片

  [D] 其他/混合
      描述其他来源或混合格式
```

---

## 意图补充说明

当用户描述意图时，使用以下引导：

| 用户说 | 引导确认 |
|-------|---------|
| "帮我做成技能" | 确认分析领域和目的 |
| "这个分析方法能用吗" | 确认是哪种分析方法（归因/趋势/探索等） |
| "能复用吗" | 确认复用场景（同类指标/同类业务/同类问题） |
| "做个模板" | 确认模板的使用范围 |

---

## 输入汇总格式

确认完成后，汇总给用户：

```
## 蒸馏目标确认

**分析领域**：{领域}
**分析目的**：{目的类型}
**文档来源**：
  - 飞书文档：{N} 个链接
  - 图片/截图：{N} 张
  - 其他：{如有}

**预期输出**：符合 skill_mp schema 的 Skill 草稿

确认开始蒸馏？[确认/补充信息/取消]
```
