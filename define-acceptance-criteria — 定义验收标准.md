# 定义验收标准

## 触发条件
用户说"帮[故事/功能]写验收标准"或要求AC。

## 输入
- 用户故事或功能描述
- （可选）需要覆盖的边界情况、技术约束

## 步骤
1. 将功能拆分为可测试的行为
2. 用假设/当/那么格式编写每条标准
3. 确保每个故事至少2条标准
4. 覆盖正常路径、异常状态和边界条件

## 输出
每个用户故事的假设/当/那么格式验收标准，覆盖边界情况。
