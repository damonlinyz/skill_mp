# 撰写功能需求

## 触发条件
用户说"写一下[功能想法]的需求"或描述一个想要文档化的功能。

## 输入
- 功能想法或描述（任何详细程度）
- （可选）目标受众、技术栈、现有产品背景

## 步骤
1. 明确要解决的核心问题
2. 撰写问题陈述（2-3句，非技术友好）
3. 起草用户故事（作为/我想要/以便格式）
4. 定义验收标准（每个故事用假设/当/那么格式）
5. 识别边界情况（空状态、异常、权限、并发）
6. 补充技术考量和待确认问题
7. 明确声明非目标

## 输出
结构化功能需求：问题陈述 + 用户故事 + 验收标准 + 边界情况 + 技术说明 + 待确认问题 + 非目标。
