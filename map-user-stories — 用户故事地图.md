# 用户故事地图

## 触发条件
用户说"梳理[功能/旅程]的用户故事地图"或要求故事地图。

## 输入
- 功能领域或用户旅程
- （可选）用户画像、发布时间线、优先级框架

## 步骤
1. 识别用户旅程阶段（列）
2. 列出每个阶段的故事
3. 按MoSCoW优先级排列（必须/应该/可以/暂不做）
4. 为每个故事添加验收标准
5. 如有时间线则按版本组织

## 输出
故事地图：按用户旅程阶段组织，按发布优先级排列，每个故事附验收标准。
